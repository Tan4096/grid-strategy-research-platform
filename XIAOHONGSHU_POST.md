# 开源了一个网格策略研究平台：回测、参数优化、实盘监测一体化

最近把自己长期在用的一套**网格策略研究工作流**整理成了一个开源项目，名字叫 **Grid Strategy Research Platform**。

这次更想从“**技术开源分享**”的角度来介绍它，而不是单纯晒界面。

GitHub 开源地址：[Grid Strategy Research Platform](https://github.com/Tan4096/Crypto_Grid_Strategy_Back_Test.git)

这个项目的定位很明确：

- 不是喊单工具
- 不是自动交易承诺
- 更不是“收益演示器”
- 它本质上是一个**面向研究和验证的策略平台**

我想解决的问题其实很朴素：

- 一套网格参数到底有没有研究价值？
- 参数优化出来的结果，是否只是过拟合？
- 回测假设和实盘状态，能不能放到一个统一界面里观察？

为了方便展示，我这次专门生成了一套 **白色背景 + 紫色主题** 的网站截图：整体延续“**最近保存**”模板的展示方式，其中回测页 K 线改成了 **2026-02-08 到 2026-03-08 的真实 BTC 行情**。

## 项目截图

### 1）回测主界面

![回测主界面](docs/assets/xiaohongshu-backtest-overview.png)

这一页是最核心的研究入口：

- 这张 K 线图使用的是 **Binance BTCUSDT 在 2026-02-08 到 2026-03-08 的真实日线数据**
- 左侧配置交易环境、区间、仓位、风险与时间范围
- 右侧同步查看收益、回撤、风险指标和 K 线 / 曲线图
- 适合快速验证一套网格策略是否值得继续深入

### 2）参数优化结果

![参数优化结果](docs/assets/xiaohongshu-optimization-results.png)

这里不是只给一个“最优参数”，而是把结果结构化展示出来：

- 参数扫描结果排序
- 最优参数摘要
- 结果回填到回测再次确认
- 不只看收益，也看回撤和稳健性

### 3）热力图分析

![热力图分析](docs/assets/xiaohongshu-optimization-heatmap.png)

热力图是我自己很常用的一种研究视角：

- 不是盯着单个冠军参数
- 而是观察**一整片参数区域**是否稳定
- 这对于避免“偶然最优”特别有帮助

### 4）实盘监测面板

![实盘监测面板](docs/assets/xiaohongshu-live-monitoring.png)

这一部分更偏“观察”和“校验”：

- 看机器人状态、持仓、挂单、成交、账单拆分
- 看收益趋势和风险状态
- 帮助把回测中的假设与实盘中的真实表现做对照

## 技术栈

这个项目目前的技术栈比较直接，没有刻意堆复杂架构：

- 前端：`React` + `TypeScript` + `Vite`
- 后端：`FastAPI`
- 图表：ECharts
- 测试：Vitest / Playwright / Pytest
- 部署：Docker Compose + systemd 模板

## 这个项目适合谁

我觉得会比较适合下面这几类人：

- 在做量化 / 策略研究工具的人
- 想自己搭一套回测 + 优化 + 监测闭环的人
- 对交易系统前后端架构、状态管理、图表交互感兴趣的人
- 想看一个“从自用工具整理成公开仓库”的真实案例的人

## 我为什么决定开源

一方面，是因为这套工具已经不只是 demo，而是我自己会持续使用和迭代的项目。

另一方面，我越来越觉得：

一个交易研究工具真正有价值的地方，不是把结果包装得多漂亮，而是：

- 逻辑能不能解释清楚
- 风险能不能看见
- 参数能不能复现
- 结果能不能被再次验证

所以这次我更希望把它作为一个**技术项目**和**研究平台**去分享，而不是“某种神奇策略展示页”。
